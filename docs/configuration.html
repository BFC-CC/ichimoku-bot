Validation Rules Summary

Enforced by ConfigLoader._validate() at load time. The bot will not start if any rule is violated.

RuleError Message
tenkan < kijun < senkou_bIchimoku periods must be strictly ascending
signal_mode in valid setMust be one of: tk_cross, chikou_cross, kumo_breakout, full_confirm
lot_mode in valid setMust be one of: risk_pct, fixed, compound
stop_loss.method in valid setMust be one of: kijun, atr, cloud_edge, fixed_pips
take_profit.method in valid setMust be one of: ratio, next_cloud, fixed_pips
0 < risk_per_trade_pct ≤ 5Risk per trade out of range
rr_ratio ≥ 1.0Risk:reward ratio too low
target_profit_pct > 0Target profit must be positive
pairs not emptyPairs list must not be empty

v1: config.yaml

The Signal Bot uses YAML configuration with ${ENV_VAR} expansion.

general:
  log_level: INFO        # DEBUG | INFO | WARNING | ERROR
  dry_run: true          # true = log only, no Discord messages

ichimoku:
  tenkan_period: 9
  kijun_period: 26
  senkou_b_period: 52
  displacement: 26
  chikou_shift: 26

signals:
  cooldown_minutes: 30   # per (pair, timeframe, signal_type)
  cloud_filter: true     # TK crosses require price on correct side of cloud
  strong_signal_only: false

pairs:
  - symbol: EURUSD
    timeframes: [H1, H4]
    enabled_signals:
      - tk_cross         # maps to tk_cross_up + tk_cross_down
      - kumo_breakout    # maps to kumo_breakout_up + kumo_breakout_down
      - chikou_cross     # maps to chikou_cross_up + chikou_cross_down

backtest:
  from_date: "2024-01-01"
  to_date:   "2024-12-31"
  output_dir: "./backtest_results"

discord:
  webhook_url: ${DISCORD_WEBHOOK_URL}   # expanded from .env
