Tool & CLI Usage Guide

All commands, flags, test patterns, and utilities

trade_bot.py — Trading Bot v3

The main automated trading orchestrator. Scans pairs, evaluates Ichimoku signals, executes trades, manages positions.

python trade_bot.py [OPTIONS]
FlagDefaultDescription
--config PATHconfig/strategy_config.jsonPath to the JSON strategy config file
--simoffForce simulation mode (no MT5 connection needed). Generates synthetic OHLCV data.
--onceoffRun exactly one cycle and exit. Useful for testing the full pipeline.
--dashboard-url URLemptyRemote dashboard URL to push state to via StatePusher
--dashboard-secret TOKENemptyBearer token for dashboard auth

Common Patterns

# Quick pipeline test (sim, one cycle, exit)
python trade_bot.py --sim --once

# Continuous simulation with logging
python trade_bot.py --sim

# Production with live MT5
python trade_bot.py

# Custom config file
python trade_bot.py --config my_config.json

# Split deployment (bot here, dashboard on Render)
python trade_bot.py --dashboard-url https://ichibot.onrender.com --dashboard-secret abc123
Cycle Interval
The bot polls every scheduler.bar_check_interval_sec seconds (default: 60). On each cycle: fetch bars → compute Ichimoku → evaluate signal → apply filters → calculate lot → execute → manage positions → poll trade events.

bot.py — Signal Bot v1

Discord notification bot. Detects 6 Ichimoku signal types and sends rich embeds.

python bot.py [OPTIONS]
FlagDefaultDescription
--dry-runfrom configForce dry run (log signals, don't send Discord messages)
--config PATHconfig.yamlPath to YAML config file
# Dry run (no Discord messages)
python bot.py --dry-run

# With custom config
python bot.py --config dev.yaml

# Production (set dry_run: false in config.yaml first)
python bot.py

run_backtest.py — Backtest Engine

Replays candles through the v1 signal pipeline. Zero look-ahead bias — uses the exact same code path as live.

python run_backtest.py [OPTIONS]
FlagDefaultDescription
--from YYYY-MM-DDfrom configStart date for backtest
--to YYYY-MM-DDfrom configEnd date for backtest
--symbol SYMBOLall configuredBacktest a single pair (e.g., EURUSD)
--tf TIMEFRAMEfrom configTimeframe (e.g., H1, H4)
--dry-runoffSkip CSV output (console only)
# Full backtest from config
python run_backtest.py

# Specific date range
python run_backtest.py --from 2024-01-01 --to 2024-12-31

# Single pair, single timeframe
python run_backtest.py --symbol EURUSD --tf H1

# Console only, no CSV
python run_backtest.py --dry-run
Synthetic Data Fallback
When MT5 is unavailable (Linux without Wine bridge), the backtest automatically falls back to make_synthetic_candles() — a deterministic random walk seeded by symbol name. Minimum 78 warmup candles (52 for Senkou B + 26 displacement) must elapse before signals fire.

Testing

375 tests, no MT5 required. All broker calls are mocked.

Run All Tests

source venv/bin/activate
pytest tests/ -v

Run Specific Test Files

# v1 signal detection
pytest tests/test_signals.py -v

# v3 signal engine (4 modes)
pytest tests/test_signal_engine.py -v

# Signal scoring
pytest tests/test_signal_scoring.py -v

# Config validation
pytest tests/test_config_loader.py -v

# Risk management
pytest tests/test_lot_calculator.py tests/test_risk_manager.py tests/test_sltp_builder.py -v

# Position management
pytest tests/test_position_manager.py tests/test_break_even_manager.py -v

# Filters
pytest tests/test_session_filter.py tests/test_news_filter_v2.py tests/test_trend_filter.py -v

# Health & monitoring
pytest tests/test_health_monitor.py tests/test_keep_alive.py -v

# Validation layers
pytest tests/test_adversarial_validator.py tests/test_momentum.py -v

pytest tests/validation/ -v

Run by Keyword

pytest tests/ -v -k "tk_cross"

pytest tests/ -v -k "chikou"

Dashboards

v3 FastAPI Dashboard (port 8000)

Starts automatically with trade_bot.py. Live WebSocket updates every 2 seconds.

URLContent
http://localhost:8000HTML dashboard: account, signals, positions, trades, verification, logs
http://localhost:8000/api/healthJSON: {"status": "ok", "uptime_sec": ..., "connected_since": ...}
http://localhost:8000/api/validation/metricsJSON: adversarial validation counters (when enabled)
ws://localhost:8000/wsWebSocket: full state snapshot pushed every 2 seconds

v1 Streamlit Dashboard (port 8501)

streamlit run app.py   # http://localhost:8501

Plotly-rendered Ichimoku charts with synthetic data. Sidebar navigation. Reads/writes config.yaml live.

Utilities

Verifies a deployed dashboard is responding and rendering correctly.

start_mt5_bridge.py

Starts the mt5linux RPyC bridge for Wine-based MT5 communication on Linux.

python3 start_mt5_bridge.py    # listens on localhost:18812

Must remain running in a separate terminal while the bot is active.

setup_env.sh

One-shot environment bootstrap. Installs system packages, creates venv, installs pip dependencies.

bash setup_env.sh

setup_mt5_wine.sh

One-time Wine setup for MT5 on Linux. Installs Wine, .NET Framework, and the MT5 terminal.

bash setup_mt5_wine.sh

Environment Variables

VariableUsed ByDescription
DISCORD_WEBHOOK_URLbot.py (v1)Discord webhook URL for signal notifications
DASHBOARD_SECRETdashboard/app.pyBearer token for POST /api/state
Secrets Management
Secrets live in .env (loaded by python-dotenv). The v1 config.yaml supports ${DISCORD_WEBHOOK_URL} expansion via os.path.expandvars. The v3 config uses plain JSON with no variable expansion — sensitive values (account password) go directly in the JSON or are passed via CLI flags.
