Architecture

System design, pipeline flow, and component relationships

Two Coexisting Systems

The repository contains two independent trading systems. They share the core Ichimoku calculation code but have separate entry points, configs, and pipelines.

Shared Components

core/indicator.py core/candle_buffer.py — Both systems use the same pure-pandas Ichimoku calculator and candle buffer. No other source files are shared.

Core Components

FileResponsibility
trade_bot.pyTradeBotOrchestrator — main loop. Each cycle: update account → check halt → process each symbol → manage positions → poll trade events → update state.
core/config_loader.pyParse JSON into typed dataclasses (Config root, 14 nested configs). Full validation on load.
core/candle_close_guard.pyStrip live bar (df.iloc[:-1]). Track last processed bar per symbol to avoid duplicate evaluation.
core/ichimoku_calculator.pyWraps indicator.py. Returns frozen IchimokuValues dataclass (immutable, thread-safe). Also provides is_chikou_clear() and pip_size().
core/signal_engine.py4 signal modes via evaluate(ichi, df). Returns SignalResult (BUY/SELL/NEUTRAL + score + conditions_met). Also check_exit() for open positions and classify_signal_strength().

Signal Modes

ModeSpeedBUY Conditions
tk_cross Fastest TK cross up + price above cloud + (opt) chikou clear + (opt) bullish cloud
chikou_cross Moderate Chikou crosses above close[-26] + chikou above cloud at -26 + price above cloud
kumo_breakout Moderate Price breaks above cloud (prev bar was inside/below) + (opt) future cloud bullish
full_confirm Conservative Price > cloud + TK alignment + chikou clear + bullish cloud + min cloud thickness

SELL conditions are the exact mirror of BUY in all modes.

Trade Execution

FileResponsibility
core/lot_calculator.py3 lot modes: risk_pct (default), fixed, compound. JPY-aware pip size. Score scaling: max(signal_score, 0.1). Rounds to volume_step, clamps to [volume_min, volume_max].
core/sltp_builder.pySL methods: kijun, atr, cloud_edge, fixed_pips (+ buffer_pips). TP methods: ratio, next_cloud, fixed_pips. Minimum SL distance: 2 pips.
core/order_executor.pySend market orders with retry (retry_attempts, retry_delay_ms). Magic number isolation on all requests.
core/mt5_connector.pyMT5 bridge + simulation fallback. force_sim=True in tests. Provides: connect, get_bars, get_account_info, get_open_positions, get_symbol_info, get_deal_history.

Position Lifecycle

OPEN → [each cycle on closed bar]
  ├── Break-even: profit_pips ≥ trigger_pips → SL = entry + lock_in_pips
  ├── Trailing stop: kijun or fixed method, min trail_step_pips improvement
  ├── Ichimoku exits: TK cross against / price enters cloud / chikou cross
  └── Virtual TP (if enabled): close manually at TP price
CLOSE → TradeEventListener.poll() → ActionVerifier.verify() → log
FileResponsibility
core/break_even_manager.pyMove SL to entry + lock_in_pips after trigger_pips profit.
core/position_manager.pyTrailing stop (kijun or fixed) + ichimoku exit conditions. Only updates SL if improvement ≥ trail_step_pips.
core/trade_event_listener.pyPoll MT5 deal history for close events. Trigger ActionVerifier + TradeLogger.
core/action_verifier.pyScore trades CORRECT/INCORRECT. Classify failures: SL_HIT, AGAINST_D1_TREND, WEAK_SIGNAL, SESSION_ANOMALY, OVERTRADED, SYSTEM_ERROR. Post-trade quality checks (slippage, fill ratio, spread).

Health Monitoring

core/health_monitor.py tracks:

  • Tick gaps — alerts if no tick received for > max_tick_gap_sec (default 300s)
  • Consecutive errors — alerts after max_consecutive_errors (default 3)
  • Execution failures — failed order attempts
  • Heartbeat — periodic "alive" signal every heartbeat_interval_sec (default 3600s)

Alerts fire via the configured notifier_fn with cooldown (alert_cooldown_sec, default 900s) to prevent spam.

Key Data Contracts

IchimokuValues (frozen dataclass)

@dataclass(frozen=True)
class IchimokuValues:
    tenkan, kijun, senkou_a, senkou_b, chikou, close       # current bar
    cloud_top, cloud_bottom                                  # derived
    prev_tenkan, prev_kijun, prev_senkou_a, prev_senkou_b   # previous bar
    prev_chikou, prev_close, prev_cloud_top, prev_cloud_bottom
    future_span_a, future_span_b                             # projected +26
    cloud_thickness_pips: float
    bar_time: pd.Timestamp

SignalResult

@dataclass
class SignalResult:
    signal: Signal           # BUY / SELL / NEUTRAL
    mode_used: str           # which mode fired
    reasons: list[str]       # human-readable conditions met
    conditions_met: dict     # {"price_above_cloud": True, ...}
    bar_time: pd.Timestamp
    ichi: IchimokuValues
    score: float = 1.0       # signal scoring (0-1)
    score_breakdown: dict    # per-component scores
    strength: str = ""       # STRONG / MODERATE / WEAK
    momentum_score: float    # 0-100

SLTPResult

@dataclass
class SLTPResult:
    sl: float          # stop loss price
    tp: float          # take profit price
    sl_pips: float     # SL distance in pips
    tp_pips: float     # TP distance in pips
    sl_method: str     # kijun / atr / cloud_edge / fixed_pips
    tp_method: str     # ratio / next_cloud / fixed_pips

Key Constraints

  • Ichimoku needs ≥78 candles for valid values (52 for Senkou B + 26 for displacement). Buffer seeds with 300.
  • Chikou is current close shifted back 26 bars. is_chikou_clear() checks high-low range, not just close[-26].
  • All signal crossovers compare prev_* vs current (two-bar edge detection).
  • CandleCloseGuard always strips the live bar (df.iloc[:-1]). Signals never evaluate incomplete bars.
  • IchimokuValues is frozen (immutable) for thread safety between bot loop and dashboard.
  • BotState uses threading.Lock for all reads/writes.
  • MT5Connector uses force_sim=True in all tests — never connects to a real broker in CI.
  • Lot score floor: max(signal_score, 0.1) ensures lot size is never zero when score scaling is enabled.
  • JPY pip size: 0.01 vs 0.0001 for all other pairs. Determined by "JPY" in symbol.
  • Magic number isolation: all MT5 queries filter by execution.magic_number. Never touches other EAs' orders.
  • failed_actions.csv uses filelock for thread-safe writes.
